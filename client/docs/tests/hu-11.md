# HU-11 — Historial de temperatura con auditoria · Checklist HMI

Casos CP-11-A y CP-11-B del Plan de Pruebas v2.0 que validan que el HMI permite
consultar el historial de temperatura por cuarto y rango, y exportarlo como CSV valido
para auditoria HACCP.

> Endpoint backend: `GET /api/cuartos/{id}/historial?desde=&hasta=&formato=json|csv`
> Ruta HMI: `/historial` — visible solo para supervisor en el header.
> Activar mock: `VITE_USE_MOCK_API=true` en `client/.env.local`.
> Credenciales: `jperez/1234` (operador), `alopez/5678` (supervisor).

---

## CP-11-A · Supervisor consulta historial de 24 h y verifica campos

**Tipo:** Funcional
**Precondicion:** Sistema corriendo al menos 2 minutos con lecturas en `lecturas_temperatura`.
HMI logueado como `alopez` (supervisor).

### Pasos

1. Verificar que en el Header aparece el link **Auditoria** (no debe aparecer para `jperez`).
2. Hacer click en **Auditoria** — navegar a `/historial`.
3. Seleccionar **Cuarto 1** en el selector de cuarto.
4. Seleccionar **Ultimas 24 h** en los radio buttons de rango.
5. Hacer click en **Consultar**.
6. Verificar que la tabla muestra registros con las columnas:
   - **Timestamp** en formato `DD/MM/YYYY HH:MM:SS`.
   - **Cuarto** con valor `1`.
   - **Temp (°C)** con signo + para valores positivos.
   - **Estado alarma** con badge de color (normal=verde, preventiva=ambar, critica=rojo).
7. Verificar el contador de registros en la parte inferior (ej. "1–50 de 2 880").
8. Navegar a la pagina 2 con el boton `›` y verificar que los timestamps avanzan.

**Resultado esperado:** la tabla se llena con lecturas reales del cuarto 1 en las ultimas 24 h.
Los campos coinciden con los del contrato E7.

## CP-11-B · Exportacion CSV abre en Excel con campos correctos

**Tipo:** Funcional + integracion
**Precondicion:** CP-11-A completado. Tabla con registros visibles.

### Pasos

1. Hacer click en **Exportar CSV** (boton verde, visible solo si hay resultados).
2. Verificar que el navegador descarga un archivo con nombre `historial_cuarto1_YYYY-MM-DD.csv`.
3. Abrir el archivo en Excel o LibreOffice Calc.
4. Verificar que la primera fila es la cabecera: `timestamp,cuarto_id,temperatura,estado_alarma`.
5. Verificar que los valores de la columna `temperatura` son numericos (no string).
6. Verificar que `estado_alarma` solo contiene `normal`, `preventiva` o `critica`.
7. Verificar que el numero de filas de datos coincide con el total mostrado en la tabla del HMI.

**Resultado esperado:** el CSV es valido, abre sin errores de encoding en Excel y los campos
coinciden con lo esperado para una auditoria HACCP.

## CP-11-B-2 · Inmutabilidad verificada en base de datos (Backend + BD)

**Tipo:** Seguridad de datos
**Precondicion:** Acceso a pgAdmin con el usuario `sei_user`.

### Pasos

1. Conectarse a PostgreSQL con el usuario `sei_user` (el que usa el backend en produccion).
2. Ejecutar: `UPDATE lecturas_temperatura SET temperatura = 99 WHERE id = 1;`
3. Verificar que el comando es **rechazado** (error de permisos).
4. Ejecutar: `DELETE FROM lecturas_temperatura WHERE id = 1;`
5. Verificar que el comando es **rechazado** (error de permisos).

**Resultado esperado:** ambas operaciones fallan con error de permisos, confirmando que los
registros de auditoria son inmutables.

**Estado:** [ ] Pass  [ ] Fail  Fecha: ___

> Nota: este caso lo ejecuta Dev Backend (T-11-04). HMI solo verifica que los datos
> devueltos por la API son consistentes con la BD.

## Verificaciones complementarias del HMI

### Rango 30 dias cambia el intervalo de muestreo en mock
1. Con `VITE_USE_MOCK_API=true`, seleccionar rango **Ultimos 30 dias**.
2. Consultar — verificar que el numero de registros es ~1 440 (cada 30 min).
3. Cambiar a **Ultimas 24 h** — verificar ~2 880 registros (cada 30 s).

**Estado:** [ ] Pass  [ ] Fail

### Error de red muestra mensaje descriptivo
1. Apagar el backend Java (o cambiar `VITE_API_URL` a una URL incorrecta) con `VITE_USE_MOCK_API=false`.
2. Intentar consultar.
3. Verificar que el HMI muestra el mensaje de error en rojo, sin romper la pagina.

**Estado:** [ ] Pass  [ ] Fail

### Paginacion funciona correctamente
1. Consultar rango 24h con cualquier cuarto.
2. Navegar a la ultima pagina con `»`.
3. Usar `«` para volver a la primera.
4. Verificar que los botones `‹` y `›` se deshabilitan en los extremos.

**Estado:** [ ] Pass  [ ] Fail

## Casos fuera de alcance del HMI

- Endpoint REST `GET /api/cuartos/{id}/historial` (T-11-01, Dev Backend).
- Rangos predefinidos en `HistorialService` (T-11-02, Dev Backend).
- Generacion de CSV en el backend (T-11-03, Dev Backend).
- Script SQL de inmutabilidad `REVOKE UPDATE, DELETE` (T-11-04, Dev Backend).
