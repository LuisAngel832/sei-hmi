# HU-08 — Aumento de potencia de refrigeracion al abrir una puerta · Checklist HMI

Caso de prueba CP-08 que valida que el HMI refleja en menos de 2 segundos la subida de
potencia al 100% con motivo `PUERTA_ABIERTA` cuando el backend publica en
`sei/cuartos/{n}/refrigeracion/estado`, y que restaura correctamente al cerrar la puerta.

> Topico de estado publicado por backend: `sei/cuartos/{n}/refrigeracion/estado`
> Payload: `{ cuarto_id, potencia_pct, motivo: NORMAL|PUERTA_ABIERTA|FORZADO_MANUAL, timestamp }`
> Credenciales de referencia: `jperez/1234` (operador), `alopez/5678` (supervisor).

---

## CP-08-A · Potencia sube al 100% en menos de 2 segundos al abrir una puerta

**Tipo:** Funcional + tiempo de respuesta
**Precondicion:** Backend Spring Boot activo, broker EMQX corriendo. HMI logueado con cualquier rol.
Cuarto 2 en estado normal (potencia ~60%).

### Pasos

1. Desde el simulador Python, publicar apertura de puerta del cuarto 2:
   ```
   sei/cuartos/2/puerta  →  { "estado": "abierta", "origen": "manual" }
   ```
2. El backend debe detectar el evento y publicar en menos de 2s:
   ```
   sei/cuartos/2/refrigeracion/estado  →  { "cuarto_id": 2, "potencia_pct": 100, "motivo": "PUERTA_ABIERTA", "timestamp": "..." }
   ```
3. En la tarjeta del cuarto 2 del HMI verificar, antes de que pasen 2 segundos desde el paso 1:
   - La barra de **REFRIGERACIÓN** llega al 100% y cambia de color cyan a **rojo**.
   - Aparece el badge **PUERTA ABIERTA** en rojo junto al label REFRIGERACIÓN.
   - El valor numerico muestra **100%** en rojo.
4. En la seccion PUERTA de la tarjeta verificar que el indicador refleja "Abierta" en ambar.

**Resultado esperado:** el HMI muestra la subida de potencia con icono diferenciado en menos de 2s.
No se requiere accion del operador.

## CP-08-B · Potencia regresa al valor anterior al cerrar la puerta

**Tipo:** Funcional
**Precondicion:** CP-08-A completado. Cuarto 2 con potencia=100, motivo=PUERTA_ABIERTA.

### Pasos

1. Desde el simulador, publicar cierre de puerta del cuarto 2:
   ```
   sei/cuartos/2/puerta  →  { "estado": "cerrada", "origen": "automatico" }
   ```
2. El backend debe publicar la restauracion de potencia:
   ```
   sei/cuartos/2/refrigeracion/estado  →  { "cuarto_id": 2, "potencia_pct": 60, "motivo": "NORMAL", "timestamp": "..." }
   ```
3. En la tarjeta del cuarto 2 verificar:
   - La barra de **REFRIGERACIÓN** vuelve al valor previo (~60%) y recupera el color **cyan**.
   - El badge **PUERTA ABIERTA** desaparece.
   - El valor numerico muestra **60%** en cyan.
4. En pgAdmin confirmar que existe un registro en `refrigeracion_estado` con:
   - `cuarto_id=2`, `potencia_porcentaje=100`, `motivo=PUERTA_ABIERTA` (evento de apertura).
   - Un segundo registro con `potencia_porcentaje=60`, `motivo=NORMAL` (restauracion).

**Resultado esperado:** la tarjeta recupera el aspecto normal, sin rastro del motivo PUERTA_ABIERTA.

**Estado:** [ ] Pass  [ ] Fail  Fecha: ___

## CP-08-C · Badge FORZADO_MANUAL visible para forzar_refrigeracion

**Tipo:** Funcional (rol supervisor)
**Precondicion:** HMI logueado como `alopez` (supervisor). Cuarto 4 con temperatura > 3 C.

### Pasos

1. Hacer click en **Forzar Refrigeracion** en la tarjeta del cuarto 4 y confirmar el dialogo.
2. El bridge publica en `sei/cuartos/4/refrigeracion/cmd`.
3. El backend procesa el comando y publica:
   ```
   sei/cuartos/4/refrigeracion/estado  →  { "cuarto_id": 4, "potencia_pct": 100, "motivo": "FORZADO_MANUAL", ... }
   ```
4. En la tarjeta del cuarto 4 verificar:
   - La barra llega al 100% con color **ambar**.
   - Aparece el badge **FORZADO** en ambar.

**Resultado esperado:** el badge diferencia visualmente un forzado manual de una apertura de puerta.

**Estado:** [ ] Pass  [ ] Fail  Fecha: ___

## Verificaciones complementarias del HMI

### Bridge descarta refrigeracion/cmd correctamente
1. Verificar en los logs del bridge (`node index.js`) que los mensajes en `sei/cuartos/+/refrigeracion/cmd`
   no generan evento socket `refrigeracion`.
2. La tarjeta del HMI NO cambia al publicar en el topico `/cmd`.

**Estado:** [ ] Pass  [ ] Fail

### Snapshot inicial con potencia correcta
1. Con el broker activo y cuarto 3 en estado `PUERTA_ABIERTA`, recargar el navegador.
2. Al reconectarse, el bridge debe emitir el evento `refrigeracion` en el snapshot.
3. La tarjeta del cuarto 3 muestra directamente la barra roja al 100% sin esperar al siguiente tick.

**Estado:** [ ] Pass  [ ] Fail

## Casos fuera de alcance del HMI

- Logica de temporizador y restauracion de potencia (T-08-02 y T-08-03, backend Java).
- Entidad JPA `RefrigeracionEstado` y persistencia en BD (T-08-01, Dev Control / Backend).
- Publicacion del evento de apertura de puerta (simulador Python).
